# Гайд по развертыванию на Localhost: Решение проблем и Инструкция для ИИ

## 1. В чем была проблема? (Ликбез для пользователя)

Когда вы запускаете сайт локально, вы можете столкнуться с двумя основными препятствиями:

*   **Конфликт портов (Port Collision):** Программы часто пытаются занять один и тот же "канал" (порт), например `:5173`. Если он занят другим процессом, сервер переходит на следующий свободный (`:5174`). Если ваши ссылки или настройки ИИ были жестко привязаны к старому порту — связь обрывается.
*   **Безопасность браузера (CORS/Secure Context):** Некоторые современные среды (особенно облачные IDE или интеграции внутри других приложений) блокируют запросы к слову `localhost` из соображений безопасности.
*   **Решение:** Использование флага `--host`. Он заставляет сервер слушать не только внутреннюю петлю (127.0.0.1), но и реальный IP-адрес вашего компьютера в сети. Это делает соединение более стабильным и "прозрачным" для браузера.

## 2. Инструкция для Нейросети (Скопируйте это ИИ)

> **Задача:** Развернуть фронтенд-сайт на локальном сервере (localhost).
>
> **Технические требования:**
> 1. **Инициализация:** Работай в папке проекта. Если используется Vite, убедись, что в `package.json` скрипт запуска содержит флаг `--host`: `"dev": "vite --host"`.
> 2. **Запуск:** Используй `npm run dev`. 
> 3. **Мониторинг:** После запуска внимательно прочти вывод терминала. Не полагайся на дефолтный порт `5173`. Найди строку `Local:` или `Network:` и используй именно тот порт, который выделила система.
> 4. **Приоритет Network URL:** Если среда выполнения имеет ограничения безопасности, используй IP-адрес из строки `Network` (например, `http://192.168.1.10:5173/`) вместо `http://localhost:5173/`. Это гарантирует обход блокировок.
> 5. **Проверка ассетов:** Убедись, что все пути к картинкам и стилям начинаются от корня проекта `/`, чтобы Vite корректно обрабатывал их в режиме dev-сервера.

## 3. Быстрый запуск для Вас

Если вы захотите запустить текущий проект снова:

1. Откройте терминал в папке `medical-orchestra-landing`.
2. Введите `npm run dev`.
3. Посмотрите на список ссылок. Если `localhost` выдает ошибку в браузере, кликните по ссылке в строке **Network**.
